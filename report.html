问题 1 - 特征观察¶

  • 'LSTAT' 是指该地区有多少百分比的房东属于是低收入阶层（有工作但收入微薄）；
  • 'PTRATIO' 是该地区的中学和小学里，学生和老师的数目比（学生/老师）。

凭直觉，上述三个特征中对每一个来说，你认为增大该特征的数值，'MEDV'的值会是增大还是减小呢？每一个答案都需要你给出理由。

问题 1 - 回答：¶

从第一步中作图得知： 1、随着'RM'增大，'MEDV'会增大，住宅平均房间数的增加，会使得房屋面积增加，自然会增加房屋的价格。

2、随着'LSTAT'增大，'MEDV'会减小，区域中被认为是低收入阶层人数越多，说明房屋所在地段的房屋普遍价值越低，只有这样的房屋低收入阶层才能购买得起，所以随着低收入阶层比例的增大，该区域房屋价格会下降。

3、随着'PTRATIO'增大，'MEDV'的变动没有明显的趋势。镇上学生与教师的数量对房屋价格的影响较小，因为房屋贵的区域也会有教育机构，从而增加学生和教师的数量，房屋便宜的地区也会有这样的教育机构，所以我们很难利用因素来判断房屋的价格。

编程练习3：定义衡量标准¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 R2 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

R2的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的R2 值为0还不如直接用平均值来预测效果好；而一个R2 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的R2，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

在下方代码的 performance_metric 函数中，你要实现：

  • 使用 sklearn.metrics 中的 r2_score 来计算 y_true 和 y_predict的R2值，作为对其表现的评判。
  • 将他们的表现评分储存到score变量中。

或

  • (可选) 不使用任何外部库，参考决定系数的定义进行计算，这也可以帮助你更好的理解决定系数在什么情况下等于0或等于1。

from sklearn.metrics import r2_score

def performance_metric(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    score = r2_score(y_true, y_predict)

    return score

def performance_metric2(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    y_bar = np.mean(y_true)
    y_diff_sq = []
    fy_diff_sq = []
    i = 0
    for y in y_true:
        y_diff_sq.append(pow(y-y_bar, 2))
        fy_diff_sq.append(pow(y-y_predict[i], 2))
        i += 1
    ss_tot = np.sum(y_diff_sq)
    ss_res = np.sum(fy_diff_sq)
    
    score = 1 - ss_res / ss_tot

    return score

问题 8 - 回答：¶

1、在样本量不充足的情况下，为了充分利用数据集对算法效果进行测试，将之前分好的拥有80%数据的训练集A分为k个包，顺序的讲将其中一个包作为测试集，剩下k-1个包作为训练集进行训练，为寻找最优评分模型做好划分。

2、 (1)首先遍历所有参数得到一个包含了所有参数可能组合的数组，假设有n组则参数数组为［p1, p2, p3, ... pn］。 (2)利用交叉验证将事先划分出来的训练集再次按比例划分成训练集和测试集，k折交叉验证会将原训练集分成k份，多余的数据会被就近分入前一组或后一组，尽可能均分数据，然后顺序取第1～k组作为测试集，其余数据为训练集，对模型进行训练。交叉验证，后通过对比这些参数选择出最佳结果并得到最佳的参数组合。 (3)评分函数将通过计算残差平方和方差的比率，来给模型打分，评判模型的拟合程度。 (4)遍历(1)中参数数组，形成模型m1, m2, m3, ... mn，每一个模型都进行一次完整的交叉验证，即每一个模型都用1~k个测试、训练集来进行训练和预测，每次训练和预测之后都会用评分函数计算一次评分，则每个模型都有k个评分的值［s1, s2, s3, ... sk］，这个模型的最终评分为这k个评分值的平均数（s1+s2+...+sk）/k。 (5)然后循环比较这n个不同参数模型的评分，得到最高评分的参数组合，和此时的模型。

3、'cv_result_'属性中存储了所有待遍历的参数组合。

4、交叉验证，通过用平均识别数作为衡量指标，来更好的描述识别率，因为样本平均数是期望的无偏估计，使用交叉验证得到的最终模型将拥有较好的泛化能力。如果不使用交叉验证，而仅仅用数据集单次分割为训练集和测试集来进行网格搜索，可能存在训练集的特征A对于结果的影响过于显著，而训练集的特征B对于结果的影响过于显著，导致找出的最优模型并不是整个训练集能得到的最好的预测模型。而交叉验证保证每个数据都能作为训练集也能作为测试集来评估模型的预测效果，该评估更接近真实的预测效果。

def cross_val_score(estimator, X, y, scoring = performance_metric, cv=3):
    """ 返回每组交叉验证的模型分数的数组 """
    X_data_test = []
    y_data_test = []
    X_data_train = []
    y_data_train = []
    X_split = len(X) / cv
    y_split = len(y) / cv
    
    for i in range(1,cv+1):
        X_data_test.append(X[(i - 1) * X_split : i * X_split])
        y_data_test.append(y[(i - 1) * y_split : i * y_split])
        X_train = X
        X_train = X_train.drop(X_train.index[(i - 1) * X_split : i * X_split])
        X_data_train.append(X_train)
        y_train = y
        y_train = y_train.drop(y_train.index[(i - 1) * y_split : i * y_split])
        y_data_train.append(y_train)
        
    scores = [0,0,0]
    for i in range(0,cv):
        estimator.fit(X_data_train[i], y_data_train[i])
        y_pred = estimator.predict(X_data_test[i])
        scores[i] = performance_metric(y_data_test[i], y_pred)
    scores = sum(scores) / cv
    return scores


def fit_model2(X, y):
    """ 基于输入数据 [X,y]，利于网格搜索找到最优的决策树模型"""
    from sklearn.tree import DecisionTreeRegressor
    best_score = 0
    for i in range(1,11):
        estimator = DecisionTreeRegressor(max_depth=i)
        score = cross_val_score(estimator, X, y)
        if score > best_score:
            best_score = score
            best_estimator = estimator
    
    #最优交叉验证分数对应的最优模型
    
    return best_estimator

模型健壮性¶

一个最优的模型不一定是一个健壮模型。有的时候模型会过于复杂或者过于简单，以致于难以泛化新增添的数据；有的时候模型采用的学习算法并不适用于特定的数据结构；有的时候样本本身可能有太多噪点或样本过少，使得模型无法准确地预测目标变量。这些情况下我们会说模型是欠拟合的。
